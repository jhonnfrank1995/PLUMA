# LG Design Excellence — checklist

## Filosofía (existencia)
### Por pantalla
- [ ] ¿Qué trabajo viene a hacer el usuario? (una frase)
- [ ] ¿Podría ocurrir sin pantalla (automatización, notificación)?
- [ ] ¿Qué se eliminó y por qué? (lista explícita)
- [ ] ¿Cómo se ve vacía / con error / con 10.000 elementos?

### Por botón/acción
- [ ] ¿Puede el sistema decidirlo solo con seguridad? → eliminar botón
- [ ] ¿Es reversible? Si no, ¿cómo se comunica el peso?
- [ ] ¿Qué pasa si se pulsa dos veces?

### Por flujo
- [ ] Pasos esenciales vs. pasos actuales
- [ ] ¿Qué paso existe solo por cómo está hecho el backend?

## Identidad visual (sistema)
- [ ] ¿Todo color/tipo/espaciado sale de un token definido? ¿Algún valor suelto?
- [ ] ¿Un solo acento primario por vista?
- [ ] ¿Mismo patrón visual = mismo significado en todo el producto?
- [ ] Contraste AA+, foco visible, no depender solo del color
- [ ] Estados diseñados: vacío / cargando / error / lleno / deshabilitado
- [ ] ¿La marca LG se siente sin gritar?
